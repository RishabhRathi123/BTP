The "Black Box" Problem

ACCURACY VS. RELIABILITY

Traditional Deep Neural Networks (DNNs) like VGG-19 prioritize maximizing classification accuracy (Cross-Entropy Loss). However, they act as "black boxes".

  • Unregulated Behavior: Neurons may fire inconsistently for the same class.
  • Lack of Interpretability: Hard to verify "why" a decision was made.
  • Fragility: High accuracy on training data doesn't guarantee robust internal logic.

Proposed Solution: Coverage-Guided Training

INTERNAL REGULARIZATION

We don't just check the final output. We regulate internal neuron activations during the training process itself.

ACTIVATION SIGNATURES

We construct class-specific "Signatures" (Multi-Range Coverage) that define the expected "safe" behavior for neurons.

DYNAMIC ADAPTATION

Signatures are updated dynamically using Exponential Moving Averages (EMA) to evolve with the model's weights.

System Architecture

MODIFIED VGG-19

We utilized a lightweight VGG-19 backbone with specific architectural enhancements for coverage analysis:

  • Base Model: VGG-19 (Convolutional Blocks + Max Pooling).
  • Instrumentation: Inserted "Coverage Layers" after key convolutional blocks.
  • Function: These layers capture activation vectors a i l during the forward pass without disrupting the inference flow.

Activation Signatures: Static vs. Dynamic

STATIC SIGNATURES

Computed once before training begins using a subset of data.

  • Pros: Simple to implement.
  • Cons: Can become "stale" as model weights change significantly during training.
  • Risk: Penalizing valid new behaviors learned by the network.

DYNAMIC SIGNATURES

Updated periodically (every T epochs) using a Sliding Window.

  • EMA Update: S new = α · S old + ( 1 - α ) · S cur
  • Benefit: Adapts to the evolving feature space.
  • Result: Better generalization and less rigid constraints.

The Loss Function: MRC Loss

DUAL-OBJECTIVE OPTIMIZATION

We optimize for both Accuracy and Consistency. The total loss is a weighted sum:

L total = L CE + λ cov · L coverage

Where L coverage consists of:

  1. Range Penalty: Penalizes activations outside the [min, max] percentile.
  2. Distribution Penalty: MSE between current and expected histograms.

Dynamic Training Workflow

1

INITIALIZATION

Initialize S ( 0 ) (Signatures) using a small subset of training data.

2

TRAINING STEP

Standard Forward Pass. Compute L CE and L cov based on current signature.

3

ACCUMULATION

Store activation statistics in a sliding window buffer W.

4

UPDATE (EPOCH T)

Refresh Signatures S using EMA of the sliding window.
